## 🔧 常见问题排查

### ⚠️ v3.6.0 开发中遇到的实际问题及解决方案 ⭐ **重要**

以下是开发过程中实际遇到并已解决的问题，如果您在测试中遇到类似问题，可参考此解决方案：

#### 问题 A: 前端登录后无法进入主界面 ⭐ **BLOCKING 级别**

**现象**: 登录成功后页面一片空白，浏览器控制台报错：
```
SyntaxError: The requested module 'http://192.168.1.133:5181/src/api/sessionTags.ts'
doesn't provide an export named: 'sessionTagsAPI'
```

**根本原因**:
- `SessionWorkbench.vue` 第 278 行导入: `import { sessionTagsAPI } from '@/api/sessionTags'`
- `sessionTags.ts` 只导出了5个独立函数，没有命名导出 `sessionTagsAPI`

**解决方案**: ✅ 已修复（v3.6.0-dev）
在 `/home/yzh/AI客服/鉴权/agent-workbench/src/api/sessionTags.ts` 添加命名导出：

```typescript
// Named export for convenience (used by SessionWorkbench.vue)
```

#### 问题 B: Vue 模板语法错误导致标签未显示 ⭐ **BLOCKING 级别**

**现象**: 登录后只能看到一个会话（钱七），其他 4 个会话不显示。浏览器控制台警告：
```
[Vue warn]: Property "tagId" was accessed during render but is not defined on instance
```

**根本原因**:
- Vue 3 不允许 `v-for` 和 `v-if` 在同一个元素上使用
- 原代码：
  ```vue
  <TagBadge
    v-for="tagId in session.tags.slice(0, 3)"
    v-if="getTagById(tagId)"  ← 错误！
    :key="tagId"
    :tag="getTagById(tagId)!"
    size="small"
  />
  ```

**解决方案**: ✅ 已修复（v3.6.0-dev）
使用 `<template>` 标签包裹 `v-for`，将 `v-if` 移到内层：

```vue
<!-- SessionWorkbench.vue 第 65-76 行 -->
<div v-if="session.tags && session.tags.length > 0" class="session-tags">
  <template v-for="tagId in session.tags.slice(0, 3)" :key="tagId">
    <TagBadge
      v-if="getTagById(tagId)"  ← v-if 移到这里
      :tag="getTagById(tagId)!"
      size="small"
    />
  </template>
  <span v-if="session.tags.length > 3" class="more-tags">
    +{{ session.tags.length - 3 }}
  </span>
</div>
```

**验证**: 修复后应能看到所有 5 个会话，每个会话下方正确显示彩色标签。

#### 问题 C: 统计数据显示错误（pending_manual 显示 0） ⭐ **数据一致性问题**

**现象**:
- 会话列表显示正确（5 个会话）
- 但统计卡片显示：待接入 0，服务中 1，全部 5（应该是：待接入 4，服务中 1，全部 5）

**根本原因**:
- 测试数据创建脚本只写入了会话数据 (`session:XXX`)，但没有维护 Redis 状态索引 SET
- 后端 `count_by_status()` 方法依赖 `status:pending_manual` SET 统计数量
- 这些 SET 为空，导致统计返回 0

**解决方案**: ✅ 已修复（v3.6.0-dev）
重建所有状态索引：

```python
import redis
import json

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 清空旧索引
r.delete("status:pending_manual", "status:manual_live", "status:bot_active")

# 扫描所有会话并重建索引
all_keys = r.keys("session:*")
for key in all_keys:
    data = r.get(key)
    if data:
        session = json.loads(data)
        session_name = session.get("session_name")
        status = session.get("status")
        if session_name and status:
            r.sadd(f"status:{status}", session_name)

# 验证
print(f"pending_manual: {r.scard('status:pending_manual')}")  # 应输出 4
print(f"manual_live: {r.scard('status:manual_live')}")        # 应输出 1
```

**验证**: 修复后统计卡片应显示：待接入 4，服务中 1，全部 5。

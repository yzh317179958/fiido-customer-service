## 📋 模块5: 坐席协作功能

### 5.1 功能概述

支持坐席之间协作处理复杂问题，包括会话转接、内部备注、@提醒和协助请求，提升团队效率。

**参考对标**：

- 拼多多客服：支持转接专家坐席
- 有赞客服：支持内部协作笔记

### 5.2 功能需求

#### 5.2.1 会话转接增强

**F5-1: 转接原因和备注**

在当前转接功能基础上增强：

**转接流程**：

1. 坐席A点击"转接"按钮
2. 选择目标坐席B（显示在线状态和当前会话数）
3. 填写转接原因（必填）
4. 填写转接备注（可选，给坐席B的说明）
5. 点击"确认转接"
6. 坐席B收到通知，可选择接受或拒绝

**转接原因类型**：


| 原因类型 | 说明                   | 示例                               |
| -------- | ---------------------- | ---------------------------------- |
| 专业技能 | 需要专业技能的坐席处理 | "需要技术支持坐席处理电池故障问题" |
| 语言要求 | 需要特定语言的坐席     | "客户要求英语客服"                 |
| 工作负载 | 当前坐席会话过多       | "当前会话数已满，转给空闲坐席"     |
| 客户要求 | 客户指定某个坐席       | "客户要求找上次服务的王客服"       |
| 其他     | 其他原因               | 自由输入                           |

**转接记录**：

```typescript
interface TransferRecord {
  from_agent: string              // 转出坐席
  to_agent: string                // 转入坐席
  reason: string                  // 转接原因
  note: string                    // 转接备注
  transferred_at: Date            // 转接时间
  accepted: boolean               // 是否接受
  accepted_at: Date | null        // 接受时间
}
```

**F5-2: 转接历史追踪**

在会话详情中显示完整转接历史：

**显示内容**：

```
转接历史:
1. 2025-01-26 14:30
   张客服 → 李客服（技术支持）
   原因: 需要技术支持坐席处理电池故障问题
   备注: 客户反馈电池续航突然下降

2. 2025-01-26 15:00
   李客服 → 王客服（售后专员）
   原因: 需要售后流程处理
   备注: 初步诊断需要更换电池，转售后办理
```

#### 5.2.2 内部备注系统

**F5-3: 内部备注功能**

坐席可以添加仅内部可见的备注，客户不可见。

**备注类型**：


| 类型     | 用途               | 示例                                     |
| -------- | ------------------ | ---------------------------------------- |
| 问题记录 | 记录客户问题关键点 | "客户购买的D4S，使用3个月后电池续航下降" |
| 处理过程 | 记录处理步骤       | "已指导客户检查电池连接，问题依旧"       |
| 解决方案 | 记录最终解决方法   | "建议客户寄回检测，3天内给出结果"        |
| 注意事项 | 记录特殊注意点     | "⚠️ 客户比较着急，需要优先处理"        |

**备注功能**：

- 添加备注：输入框 + "保存"按钮
- 编辑备注：点击备注内容可编辑
- 删除备注：仅创建者和管理员可删除
- 时间戳：自动记录创建时间和创建者
- @提醒：支持@其他坐席（见F5-4）

**UI设计**：

```
┌─────────────────────────────────────┐

│ 客户购买的D4S，使用3个月后电池续航   │
│ 突然从50公里降到30公里               │
│                          [编辑] [删除]│
├─────────────────────────────────────┤
│ 📝 李客服 (15:00)                    │
│ 已指导客户检查电池连接和充电器，     │
│ 问题依旧。初步判断电池损耗异常       │
│                          [编辑] [删除]│
├─────────────────────────────────────┤
│ 添加备注:                            │
│ ┌───────────────────────────────┐   │
│ │ [输入备注...] [@提醒]          │   │
│ └───────────────────────────────┘   │
│                      [取消] [保存]   │
└─────────────────────────────────────┘
```

**F5-4: @提醒功能**

在内部备注中支持@其他坐席：

**使用方式**：

1. 输入 `@` 触发提醒
2. 显示在线坐席列表
3. 选择坐席，自动插入 `@王客服`
4. 保存备注后，被@的坐席收到通知

**通知方式**：

- 浏览器通知："张客服在会话'XXX'中@了你"
- 工作台顶部红点提醒
- 点击跳转到对应会话

**示例**：

```
📝 张客服 (14:30)
@李客服 这个电池问题你帮忙看下，我不太确定是不是需要更换
```

#### 5.2.3 协助请求

**F5-5: 请求协助功能**

坐席遇到复杂问题时，可以请求其他坐席协助，但不转接会话。

**协助流程**：

1. 坐席A点击"请求协助"
2. 选择协助坐席B
3. 填写协助请求："客户询问电池保修期，我查不到政策"
4. 坐席B收到协助请求通知
5. 坐席B在协助面板中回复："保修期1年，查询方式..."
6. 坐席A收到回复通知

**协助请求数据结构**：

```typescript
interface AssistRequest {
  id: string                      // 请求ID
  session_name: string            // 会话ID
  requester: string               // 请求者
  assistant: string               // 协助者
  question: string                // 请求内容
  answer: string | null           // 回复内容
  status: 'pending' | 'answered'  // 状态
  created_at: Date                // 创建时间
  answered_at: Date | null        // 回复时间
}
```

#### 5.2.4 协作历史记录

**F5-6: 协作日志**

记录会话的所有协作活动：

**记录内容**：

- 转接记录（谁→谁，原因，时间）
- 内部备注（谁，内容，时间）
- @提醒（谁@谁，内容，时间）
- 协助请求（谁请求谁，问题，回复）

**用途**：

- 复盘会话处理过程
- 培训新坐席
- 考核坐席协作能力
- 优化流程

### 5.3 数据模型

**内部备注数据结构**：

```typescript
interface InternalNote {
```

## 📋 模块6: 工作台体验优化

### 6.2 功能需求

#### 6.2.1 快捷键系统

**F6-1: 全局快捷键**

**设计原则**：使用 `Ctrl+Shift` 双修饰键避免与浏览器原生快捷键冲突

**已实现的快捷键（v3.9.0）**：


| 快捷键           | 功能         | 说明                       |
| ---------------- | ------------ | -------------------------- |
| **导航类**       |              |                            |
| `Ctrl+Shift+f`   | 搜索会话     | 聚焦到搜索框               |
| `Ctrl+ArrowUp`   | 上一个会话   | 在会话列表中向上切换       |
| `Ctrl+ArrowDown` | 下一个会话   | 在会话列表中向下切换       |
| `Escape`         | 关闭面板     | 关闭当前打开的对话框或面板 |
| **操作类**       |              |                            |
| `Ctrl+Shift+t`   | 转接会话     | 打开会话转接对话框         |
| `Ctrl+Shift+r`   | 释放会话     | 释放当前会话               |
| **功能类**       |              |                            |
| `Ctrl+Shift+b`   | 内部备注     | 打开内部备注面板           |
| `Ctrl+Shift+/`   | 快捷命令面板 | 打开快捷命令选择器         |
| `Ctrl+Shift+?`   | 快捷键帮助   | 显示快捷键帮助面板         |

**计划实现的快捷键（P1）**：


| 快捷键           | 功能     | 说明                   |
| ---------------- | -------- | ---------------------- |
| `Ctrl+Shift+1~9` | 快捷回复 | 插入对应编号的快捷回复 |
| `Ctrl+Shift+l`   | 标签管理 | 打开标签选择器         |
| `Ctrl+Shift+h`   | 历史记录 | 查看会话历史           |
| `Ctrl+Shift+s`   | 设置     | 打开设置面板           |
| `Enter`          | 发送消息 | 发送当前输入的消息     |
| `Shift+Enter`    | 换行     | 在消息中换行           |

**技术约束**：

- ❌ 避免使用浏览器原生快捷键（Ctrl+T、Ctrl+N、Ctrl+H、Ctrl+L、Ctrl+S、Ctrl+R、Ctrl+F、Ctrl+B）
- ✅ 使用 `Ctrl+Shift` 组合避免冲突
- ✅ 在输入框聚焦时禁用导航类快捷键（通过 `allowInInput: false`）
- ✅ 特殊按键（Escape、方向键）可直接使用

**快捷键配置**：

- 用户可自定义快捷键
- 快捷键冲突检测
- 快捷键帮助面板（按 `?` 显示）

#### 6.2.3 工作状态管理

**F6-5: 坐席状态**


| 状态 | 说明                       | 显示    |
| ---- | -------------------------- | ------- |
| 在线 | 可以接入新会话             | 🟢 在线 |
| 忙碌 | 正在处理会话，暂不接新会话 | 🟡 忙碌 |
| 小休 | 短暂休息（如厕、喝水）     | 🟠 小休 |
| 午休 | 午餐时间                   | 🟠 午休 |
| 培训 | 参加培训，不接会话         | 🔵 培训 |
| 离线 | 下班或不在线               | ⚪ 离线 |

**状态切换**：

- 点击工作台右上角状态按钮
- 选择新状态
- 可填写状态说明（如："午休，13:30回来"）

**自动状态**：

- 超过5分钟无操作 → 自动切换为"忙碌"
- 检测到登出 → 自动切换为"离线"
- 工作时间结束 → 提醒切换为"离线"

## 📝 实施注意事项

### 技术约束

1. **快捷键冲突检测**：避免与浏览器原生快捷键冲突
2. **浏览器通知权限**：需要用户授权
3. **离线检测**：使用 Navigator.onLine 和心跳机制
4. **设置持久化**：localStorage，注意容量限制

### 开发建议

1. 先实现核心快捷键，再实现自定义
2. 先实现浏览器通知，再实现声音提醒
3. 使用事件总线管理通知
4. 快捷键使用成熟库（如 hotkeys-js）

### 浏览器兼容性

- Notification API：需要 HTTPS
- SSE：现代浏览器都支持
- localStorage：容量限制 5MB
